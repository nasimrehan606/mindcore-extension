# Purpose

This document defines the official icon system for the MindCore ecosystem.

Icons play a vital role in creating an intuitive, accessible, and visually consistent user experience across all MindCore products.

These guidelines apply to the Browser Extension, Website, Mobile App, Desktop App, Admin Panel, and Marketing materials.

---

# Overview

MindCore icons should feel:

• Modern

• Minimal

• Professional

• Friendly

• Consistent

• Easy to recognize

Every icon should communicate its purpose instantly without unnecessary decoration.

# Accessibility

Icons should never be the only method of communication.

Whenever possible:

- Pair icons with text labels.
- Include tooltips for icon-only buttons.
- Ensure sufficient color contrast.
- Support keyboard navigation.

---

# Icon Usage Guidelines

## Do

✓ Use icons consistently.

✓ Match icon size within the same section.

✓ Keep spacing uniform.

✓ Use semantic colors.

✓ Use outlined icons throughout the interface.

---

## Don't

✗ Mix outlined and filled styles.

✗ Stretch or distort icons.

✗ Rotate icons unnecessarily.

✗ Add excessive shadows.

✗ Replace standard icons without updating documentation.

# Design Decisions

## D-016

Decision

Lucide Icons is the official icon library for MindCore.

Reason

Provides a modern, lightweight, and consistent icon system with excellent React integration.

Impact

All standard interface icons should come from the Lucide library unless a custom icon is officially approved.

Status

Accepted

Date

2026-06-29

---

## D-017

Decision

Outlined icons are the default style.

Reason

Outlined icons create a cleaner, more modern, and consistent interface.

Impact

Avoid mixing outlined and filled icons within the same interface.

Status

Accepted

Date

2026-06-29

---

## D-018

Decision

MindCore may introduce custom icons only for unique product concepts such as Rabi, Learning Paths, and AI Memory.

Reason

Custom icons strengthen brand identity while maintaining consistency.

Impact

Any new custom icon must be documented before implementation.

Status

Accepted

Date

2026-06-29
