# Purpose

The Changelog records every significant change made to the MindCore project.

It provides a historical record of project progress, making it easy to track what changed, when it changed, and why.

This document should be updated whenever a significant change is made to the documentation, design, development, testing, or release of MindCore.

---

# Versioning Standard

MindCore follows Semantic Versioning.

```
MAJOR.MINOR.PATCH
```

Example:

```
1.0.0
```

- **MAJOR** – Breaking or major releases
- **MINOR** – New features
- **PATCH** – Bug fixes and small improvements
